# 🏕️ Nucampsite App
A mobile campsite review app built with React Native, extending the original Nucampsite concept from web to a native mobile experience. Users can browse campsites, view details, leave comments/reviews, favorite campsites, and reserve a campsite — all from their phone.

Features


🔍 Browse Campsites — Scroll through a directory of campsites with photos, descriptions, and details
⭐ Reviews & Comments — Read existing reviews and submit your own comments/ratings
❤️ Favorites — Mark campsites as favorites for quick access later
📅 Reservation System — Select dates and reserve a campsite directly in the app
🔐 Authentication — Login flow to gate features like posting reviews or making a reservation
📱 Cross-Platform — Built with React Native to run on both iOS and Android from a single codebase


Project Structure 
 android/         - Native Android project files
 ios/              - Native iOS project files
 assets/images/    - App images/icons
 components/       - Reusable UI components
 features/         - Redux slices/feature logic
 screens/          - App screens (Directory, Detail, Reservation, etc.)
 shared/           - Shared utilities/constants
 screenshots/       - App screenshots




Tech Stack


React Native — Used our React framework's component-based architecture to build a native mobile UI
Redux — Global state management for campsites, comments, reservations, and auth
React Navigation — Screen-to-screen navigation
JavaScript (ES6+)

What I Practiced

How to take our React framework and apply it to mobile development — learning that you don't need to reinvent the wheel, just modify and adapt existing components to fit the new technology (translating web component patterns into React Native equivalents)
Building a reservation system for the campsites, including date selection logic and wiring it up to app state, so users can actually book a site rather than just view it
Managing shared state across multiple screens with Redux in a mobile context
Structuring a React Native project end-to-end (navigation, screens, components, platform-specific folders)
